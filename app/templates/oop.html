OOP In A Calculator Program

Encapsulation:

Keeping attributes of a given object private from other objects. This allows us to write more scalable code without rewriting code that will be repeated.


In the Calculator object we encapsulated the mathematical operations.

Inheritance:

Creating children objects that take, or inherit, functions and attributes from their parents.


In the Calculator, we created a Calculation object which was inherited by the 4 different math operations.

Polymorphism:

Defining a parent structure that contains functions that can be used by many objects, but in different ways.


In the Calculator we used polymorphism for each of the 4 different math operations as they all had to get a result, but in a different way

Abstraction:

Extracting and summarizing the core components of an object into a presentable and usable format.


In the Calculator object, the user should not be able to directly manipulate the history, but when they do a math operation using the function we defined, that operation will automatically be added to history within the function.
